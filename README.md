# 玄问临占

一个自己使用的传统临时问事工具，前端使用 uni-app + Vue 3，先发布 H5，后续可迁移为小程序或 App。

## 当前定位

- 不做商业化，不考虑付费和拉新。
- 重点不是八字人生盘，而是临时问事。
- 已接入今日签、梅花易数、小六壬、六爻、奇门和个人盘；其中小六壬、六爻、奇门、个人盘为简化可用版，后续可继续做传统细化。
- 前端选型为 uni-app + Vue 3，方便后续发布 H5、微信小程序和 App。
- 规则计算固定在后端接口，前端只负责渲染，AI 只负责白话深断。

## 已实现

- 手机号 + 密码登录：
  - 每个用户只访问自己的数据。
  - 前端不再展示档案模块，问卜按账号维度保存和限频。
- 问卜标签：
  - 顶部可多选事业、感情、财运、合作、出行、抉择、应期等标签。
  - 业务类标签会同步梅花问事分类。
  - 标签会随今日签和梅花问事一起进入后端结果。
- 首页“今日问卜”：
  - 可结合今日所问、当前对象、地点、个人信息。
  - 后端随机摇签并返回解签。
  - 前端有摇签动画。
  - 后端按账号限制一日一签，当日再次请求会复用同一结果。
- 问卜方式说明：展示今日签、梅花易数、小六壬、六爻、奇门、个人盘分别适合的问题类型，并可直接切换起占。
- 所问之事输入。
- 事项分类：事业、感情、财运、合作、出行、其他。
- 起卦方式：
  - 时间起卦：优先按农历年支数、农历月、农历日、时辰取数。
  - 数字起卦。
- 梅花易数结果：
  - 本卦。
  - 互卦。
  - 变卦。
  - 动爻。
  - 体用。
  - 五行生克。
  - 白话断语。
- 其他问卜方式：
  - 小六壬：按月日时取数，快速判断来不来、成不成、等不等。
  - 六爻：自动摇出六爻值，展示本卦、变卦、动爻和断语。
  - 奇门：按当前时局生成简化九宫盘，给出可用门和行动建议。
  - 个人盘：按出生日期和时间生成长期背景提示。
- 本地问事记录，保存在浏览器 localStorage。
- AI 深断按钮和接口配置预留。
- AI 继续追问：围绕当前卦继续问不懂的地方，追问会话按卦保存到 localStorage。
- 问事类后端限频：同一账号一小时一次。

## AI 接入

公网部署时 AI 走后端代理，浏览器不会拿到真实 API key。复制 `.env.example` 为 `.env`，填写服务端配置：

```text
AI_BASE_URL=你的接口地址
AI_API_KEY=你的 key
AI_MODEL=你的模型名
AI_TEMPERATURE=0.72
```

当前按常见 OpenAI 风格接口调用：

```text
POST {baseUrl}/chat/completions
```

前端默认请求同源接口：

```text
POST /api/chat/completions
```

## 部署

当前 Docker 部署包含两个服务：

- `mei-hua-wenshi`：uni-app H5 构建后由 nginx 服务，对外端口 `3007`。
- `mei-hua-wenshi-ai-proxy`：后端接口服务，容器内端口 `8787`，用户数据持久化到 `./data/app-data.json`。

Cloudflare Tunnel 当前公网访问：

```text
https://wenshi.cccode.com.cn
```

## 文件说明

- `src/pages/index/index.vue`：首页页面、样式、结果渲染、历史记录、AI 请求逻辑。
- `src/pages.json` / `src/manifest.json`：uni-app 页面和平台配置。
- `index.html` / `vite.config.mjs`：H5 构建入口和 Vite 配置。
- `divination.js`：后端起卦、排卦、体用、生克、断语规则。
- `advancedDivination.js`：后端小六壬、六爻、奇门、个人盘的简化起占规则。
- `dailyOracle.js`：后端今日摇签签文库和解签规则。
- `userStore.js`：用户、会话、今日签和问事限频数据存储。
- `server.js`：后端接口、AI 代理，不把 key 暴露给浏览器。
- `package.json`：uni-app + Vue 3 前端依赖和构建脚本。
- `Dockerfile` / `Dockerfile.api` / `docker-compose.yml`：前端静态服务和 AI 代理部署配置。
